## 为什么 Windows 电脑卡顿这么难治根？

先说清楚这个问题的本质，才知道 CPUBalance 解决的是什么。

Windows 的进程调度器本身有个老毛病：它在分配 CPU 资源时，不会主动干预那些"占着资源不说话"的高负载进程。当一个程序（比如视频编码器、游戏引擎、编译器）开始大量占用 CPU，系统并不会主动限制它，只是让它跑——结果就是整台机器的响应速度都受牵连。

这不是 CPU 核心数的问题。就算你有 16 核，这个问题依然存在。

解法思路只有一个：在操作系统层面干预进程优先级，让高负载进程"让路"，而不影响它本身的工作速度。ProBalance 算法做的就是这件事。

## CPUBalance 的 ProBalance 算法是怎么工作的？

ProBalance 是 Bitsum 公司从 2002 年开始开发、经过二十多年验证的算法。CPUBalance 就是把 Process Lasso 里最核心的 ProBalance 算法单独拎出来，做成了一个更轻量、更简单的独立工具。

工作原理不复杂，分三步：

1. **实时监控**：CPUBalance 在后台持续监控所有进程的 CPU 占用情况，实时记录每个进程的资源使用比例。

2. **触发干预**：当某个进程的 CPU 占用超过设定阈值（系统总占用率触发或单进程占用率触发，两种模式可选），CPUBalance 自动介入。

3. **动态调整**：临时压低该进程的优先级，把 CPU 资源腾给你正在交互的前台程序。等这个进程的占用率恢复正常，优先级立刻还给它。

注意：不是让那个程序变慢，而是让它别把整个系统拖垮。你的 Handbrake 还在跑，转码速度基本不受影响，但鼠标终于能动了，浏览器终于能打开了。

Softpedia 的评测中提到，CPUBalance 可以对进程设置详细的日志记录，包括时间、进程名称、触发动作（如 ProBalance 限制开始、进程优先级临时降低）等信息，并支持导出为 CSV 文件。
